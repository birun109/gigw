GIGW 3.0 Statement 5.2.7: Info and Relationships (WCAG 1.3.1)

Analysis and Requirements

GIGW Statement 5.2.7. Information, structure and relationships conveyed through presentation can be programmatically determined or are available in text.
WCAG Reference Success Criterion 1.3.1 Info and Relationships
WCAG Level Level A
Clause in Short Form Text Alternatives for Structure and Relationships
Goal Ensure the semantic structure of content is available to assistive technologies, independent of its visual appearance.
Requirement Content structure, relationships, and status information must be conveyed programmatically using proper HTML semantics.

Success Examples ✅

Data Tables (Structure and Relationships)

Scenario: Presenting hierarchical data showing annual budget allocation by department.

Annual Budget Allocation (In Crores)
Department 2024 Allocation
Education 500
Health & Family Welfare 850

Compliance Justification: The use of <th> with scope attributes programmatically establishes relationships between headers and data. Screen readers announce context for each cell correctly.

Form Field Groups (Relationships)

Scenario: Grouping related controls (radio buttons) with a common title.

Select Preferred Contact Method

Compliance Justification: The <fieldset> groups the controls, and the <legend> provides programmatic group context for screen readers.

Failure Examples ❌

Visual Hierarchy (Structure)

Scenario: Creating a visual hierarchy using only styling (font size and bolding).

Project Status Report
Financial Overview

Data here...

Technical Progress

More data here...

Compliance Justification: Using <div> and <span> loses hierarchical structure for assistive technologies. Headings (<h1>-<h6>) must be used instead.

Reliance on Visual Position (Relationship)

Scenario: Associating a form field with its label only by visual proximity.

Compliance Justification: Screen readers may announce "edit text" without context. Use <label for="id"> to programmatically associate labels with input fields.

Summary of Conformity

WCAG Conformity Summary
WCAG Reference WCAG Level Key Action for Compliance
SC 1.3.1 Info and Relationships A Use semantic HTML (<h1>-<h6>, <table>, <label>, <fieldset>, etc.) to ensure content structure and relationships are available to assistive technology. Do not rely on visual presentation alone.
