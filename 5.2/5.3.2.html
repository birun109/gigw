GIGW 3.0 Statement 5.3.2: Secure Hosting Environment

Analysis and Requirements

**Requirement Focus**Confidentiality, Integrity, and Availability (CIA) of the Hosting Environment.
**Geographical Mandate**Hosting of web infrastructure must be within the geographical boundaries of India.
**Availability Requirement**DC, BCP, and DR environments configured for High Availability (HA) in geographically separate seismic zones.
**Security Tools Mandate**Hosting Service Provider (HSP) must deploy and configure a **Web Application Firewall (WAF)**.
**Policy Mandate**WIM (Website Information Manager) must define a clear policy on accessibility outside India.

Mandatory Government Organisation Actions

Contingency and Infrastructure Requirements

  • **Resilience:** Ensure HSP provides DC and DR environments that are geographically located far from each other in **different seismic zones**.
  • **Drills:** Conduct **periodic drills of the Disaster Recovery environment** (at least once a year).
  • **Hardening:** Servers and network devices must be hardened with latest security patches.
  • **Assessment:** Periodic **Vulnerability Assessment (VA) and Penetration Testing (PT)** must be performed.
  • **Logging:** Maintain logs of the ICT infrastructure for a rolling period of **180 days** as per CERT-In directions.

Developer/System Admin Actions (Hardening & Configuration)

Server and Protocol Security Checklist

  • **Access Control:** Restrict admin access, implement the principle of least privilege, and remove default/guest accounts.
  • **Network:** **Disable all unnecessary ports** (deny all access by default) and ensure network segmentation.
  • **Secure Protocols:** Mandatorily enforce HTTPS & HSTS and disable HTTP. Use **SFTP** for file transfer.
  • **SSL:** Use a **valid SSL Certificate** (at least **2048-bit SHA 256** encryption) and monitor expiry. Disable weak ciphers (SSLv2, TLS v1.0/1.1).
  • **Information Disclosure:** Configure the HTTP Service banner to **not disclose Web Server and OS type/version**.
  • **Code Isolation:** Isolate development environments from the production network.

Example for Success Criteria ✅

Success Scenario: Secure and Resilient Hosting

Scenario: The Department hosts its new application on NIC Cloud (or a certified HSP).

The hosting is confirmed to be located in Delhi (DC) and Hyderabad (DRC) - two different seismic zones. The HSP confirms a WAF is active and hardened. The System Admin enforces HTTPS/HSTS, disables all ports except 443, and the annual DR drill was successfully completed last quarter.

**Compliance Justification:** All critical elements of the mandate are met: **Geographical location in India, HA setup (DC/DRC), WAF deployment, annual DR drill, and server hardening** (HTTPS/port closure). This is fully compliant with GIGW 5.3.2.

Example for Failure Criteria ❌

Failure Scenario: Lack of Resiliency and Hardening

Scenario: The application is hosted on a single server, and the backup server is located in the same data center room.

The system admin has kept port 80 (HTTP) open for debugging purposes, and the default "Administrator" account on the web server has not been renamed. No WAF is deployed.

**Compliance Justification:** The failure to provide geographically separate DC/DR environments violates the **Availability** mandate. Keeping port 80 open and retaining default admin accounts violates the **Hardening** and **Integrity** mandates. This fails GIGW 5.3.2.

Summary of Conformity

GIGW Requirement Key Mandate
5.3.2 Secure Hosting Environment Ensure the hosting environment is resilient (geographically separate DC/DR), hardened (no unnecessary ports, HTTPS/HSTS, WAF), and operated under strict security policies (least privilege, 180-day logging).
