GIGW 3.0 Statement 5.4.10: Bilingualism and Unicode Standard

Analysis and Requirements

**Requirement Focus**Linguistic inclusivity and technical standard for content display.
**Mandate**Website/app must be bilingual and use Unicode characters.
**Usability Mandate**Language selection link must be **prominent**.
**Government Action**Prioritize translation of **widely accessed content** if full translation is infeasible.
**Developer Action**
  • Use **Unicode character set** (e.g., UTF-8).
  • Perform **thorough testing** to prevent layout issues.
**Key Benefit**Enables wider use by Indian demographics comfortable in regional languages.

Example for Success Criteria ✅

Success Scenario 1: Prominent Selection and Unicode

Action: The website of a state government is available in English and the official regional language (e.g., Marathi).

  • The link is displayed as a large button in the **top-right corner of the header**, clearly labeled "**English** / **मराठी**".
  • The developer confirms the website meta tag for character encoding is set to Unicode (UTF-8).
    • <meta charset="UTF-8">

**Compliance Justification:** The site is bilingual, the language link is prominent, and the technical foundation (Unicode) is used, ensuring correct display and avoiding 'junk' characters. This is fully compliant with GIGW 5.4.10.

Example for Failure Criteria ❌

Failure Scenario 1: Non-Prominent Link

Action: The website offers content in Hindi and English. The language switch is a small, grayed-out text link labeled "Lang" hidden in the very bottom footer of the page, or it requires the user to navigate to the "Accessibility" page to switch.

**Compliance Justification:** Although the content is bilingual, the selection link is **not prominent**. This hinders usability for users who rely on the regional language, thus failing the usability mandate of GIGW 5.4.10.

Failure Scenario 2: Non-Unicode Font Issues

Action: The developer used an outdated, proprietary font encoding for the regional language content. When the site is viewed on a mobile device that lacks that specific font, the regional language content appears as unintelligible boxes (mojibake).

**Compliance Justification:** The failure to use the **Unicode character set** (like UTF-8) leads to content that is not universally readable across different devices, resulting in **loss of layout** and illegibility. This technical failure is a violation of the developer action in GIGW 5.4.10.

Summary of Conformity

GIGW Requirement Key Mandate
5.4.10 Bilingualism and Unicode The website must be **bilingual** with a **prominent and easy-to-use language selection link**. Developers must strictly use the **Unicode character set** (e.g., UTF-8) for all regional language content to ensure universal display and prevent layout corruption.
