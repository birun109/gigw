Failure Examples – Meaningful Sequence (Logical Reading Order) ❌

Failure Example 1: Procedural Steps in Incorrect Source Order

Scenario: Multi-step registration process presented visually in order, but incorrect in source code.

Submit payment to complete registration.
Fill out the personal details form.
Verify your email address via the link sent.

Why This Fails:
The steps are not marked using an <ol> element. The source code order does not follow the logical sequence (1, 2, 3). Assistive technologies will read the payment step first, which changes the meaning of the procedure.

Failure Example 2: FAQ with Disordered Question–Answer Sequence

Scenario: FAQ where answers appear before questions in the source order.

Use your tracking ID on the 'Track Status' page.
Question: How do I track my application status?
Aadhar card and PAN card are required.
Question: What documents are required?

Why This Fails:
The answer appears before its corresponding question in the HTML source. Screen readers will announce the information before providing context. This disrupts the logical reading sequence and affects meaning.
